Developer console: F-12

var xhttp = new XMLHttpRequest(): object for a request you're about to make. fetch is the modern 
replacement. 

xhttp.onreadystatechange = function() {}: event handler. when the state of the object changes, 
the browser calls the event handler function  

inside the event handler, this refers to the object. .readyState == 4 means the request is 
done (XHR has states 0-4), .status == 200 means the server responded HTTP 200 ok 

this.responseText: a string of JSON text 
JSON.parse turns JSON text into an array you can iterate over.

JS 

functional programming uses immutable data structures. so instead of modifying an array, 
you define a new one and make your edits, i.e, 
const t = [1, -1, 3]
const t2 = t.concat(5)  // creates new array
const m1 = t.map(value => value * 2)

destructuring: 
const [first, second, ...rest] = t

note that you can access an object's property using dot or !!bracket notation!! and you can 
add properties to an object 

functions can be defined using arrow functions or the keyword function as follows: 
function product(a, b) {
    return a * b 
}
or function expression 
mul = function(a, b) {
    return a * b
}

Destructing: const {name, age} = props // name = props.name, age = props.age
we can even destructure the parameter directly rather than in the function body: 
const Hello = ({name, age}) => {}
